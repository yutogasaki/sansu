# UX Tone Polisher

子供向けUI文言のトーン監査。正解/不正解/ヒント/スキップ/エラーの表現が「やさしく、責めず、前向き」であることを確認する。

## 参照順
1. `CONSTITUTION.md` — 第1優先: 子ども/保護者の明確さと信頼
2. `docs/product/07_ui_design_guideline.md` — トーン・トークン定義
3. `docs/product/design_review_checklist.md` — レビュー観点
4. 変更差分（`git diff`）または指定ファイル

## トーンルール

### 禁止表現
- **命令口調**: 「〜しなさい」「〜しろ」→ 「〜しよう」「〜できるよ」に置換
- **責め**: 「まちがい」「ダメ」「失敗」→ 「ちょっと ちがったね」「もういちど やってみよう」
- **不安を煽る**: 「時間がない」「急いで」→ 「きょうの ペースで だいじょうぶ」
- **過剰な競争**: 「勝った！」「負けた」→ 控えめな表現
- **大人向け言葉遣い**: 子供が読める平易な日本語にする

### 推奨パターン
- 不正解: 「ちょっと ちがったね」+ 正解表示 + 「つぎへ」
- スキップ: 「とばして だいじょうぶ」+「また でてくるよ」
- 正解: 穏やかな⭕表示。過剰な祝福は避ける
- エラー: 「だいじょうぶ」+ 具体的な次の行動
- 励まし: 「むりは しなくて いいよ」「じわっと ふえてるよ」

### 1メッセージ1意図
- 1つの吹き出し/バナーに複数の意図を詰め込まない
- 長文を避け、子供が一目で読める長さにする

## 監査手順

1. `git diff` で変更されたUI文字列を抽出
2. 上記の禁止表現に該当するものがないかチェック
3. 正解/不正解/スキップ/エラーの各状態で適切なトーンか確認
4. ikimono（ふわふわ）のセリフが5つのトーン（cozy/growing/sharing/celebrating/guiding）と整合するか確認

## 出力フォーマット

```
## UX Tone Report
### Summary
- トーン整合: OK / 要修正

### Issues
1. 箇所（ファイル:行）
   - Before: 現在の表現
   - After: 推奨表現
   - 理由:

### Confirmed OK
- チェック済みで問題なかった箇所
```

## 禁止
- 好みだけで指摘しない（ルール根拠を必ず示す）
- 英語文言もやさしいトーンを維持（日本語と同じ基準）
- トーンを直す際に意味を変えない
