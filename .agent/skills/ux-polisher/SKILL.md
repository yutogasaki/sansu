# UX & Copy Polisher (sansu) - Workspace Skill

このスキルは「文言」「メッセージ」「トーン」「UX違和感」「正解/不正解メッセージ」などの話題が出たときに適用する。

## 1) ゴール
- 子どもが萎えない、安心できる文言に統一する
- 保護者にも分かりやすい表現にする
- 文言のブレ（日本語/英語、丁寧さ、語尾、責める表現）をなくす

## 2) 基本トーン（必須）
- 命令しない（〜してください → 〜できます / 〜しよう）
- 責めない（間違い → 次が分かる案内）
- 不安を煽らない（エラー → 落ち着いた再試行案内）
- 短く、やさしく、前向きに

## 3) 文言ルール
- 1メッセージ = 1意図（長文禁止）
- 子ども向けは「ひらがな多め」寄りでもOK（ただし幼すぎない）
- 保護者向け設定は「丁寧・簡潔」

## 4) 正解/不正解/ヒントの統一テンプレ
### 正解（例）
- 「いいね！せいかい！」
- 「そのちょうし！」
- 「バッチリ！」

### 不正解（例）
- 「おしい！もういちどやってみよう」
- 「だいじょうぶ。つぎでいこう」
- 「ヒントを見てみる？」

### スキップ（例）
- 「いったんとばして、あとでやろう」
- 「OK！あとでチャレンジしよう」

### エラー/通信/保存（例）
- 「うまくできなかったみたい。もういちどためしてみよう」
- 「データをほぞんできるよ」
- 「ほぞんできなかったみたい。もういちどやってみよう」

## 5) 出力フォーマット（固定）
文言改善の依頼が来たら、必ずこの形式で出す：

## Copy Audit
### Issues（気になる点）
- （責める/命令/長い/ブレなど）

### Proposed Copy（修正案）
- 画面/状況ごとに「Before → After」で列挙

### Tone Rules（今回適用したルール）
- （箇条書き）

### Consistency Check
- 正解/不正解/ヒント/スキップ/エラーでトーンが揃っているか

## 6) 英語メッセージがある場合
- 日本語と同じトーン（優しい/短い/前向き）に揃える
- 直訳で冷たくしない
- 例：
  - Correct! → Nice! / Great job!
  - Incorrect. → Almost! Try again.
  - Error. → Oops… Try again.

## 7) 禁止
- 恥をかかせる/責める/否定する表現
- 不安を煽る表現（「失敗しました」「致命的」など）
- 一度に大量の言い回しバリエーションを盛りすぎる（運用できない）
