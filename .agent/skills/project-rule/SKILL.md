# Sansu Project Rule（Workspace Skill）

このスキルは「算数アプリ（sansu）」に関わる作業のときに適用する。
AIはタスク内容から本スキルが有効と判断した場合、以下のルールに従うこと。

## 1) 目的（このプロジェクトの前提）
- 対象：子ども（学習者）＋保護者（管理/見守り）
- ゴール：迷わず学習でき、つまずいても嫌いにならないUX
- 優先順位：分かりやすさ > 安定性 > 保守性 > 速度 > 新奇性

## 2) 仕様書駆動（Spec First）
- 実装前に必ず「今回反映する仕様」を明確化する
- 仕様にない変更は勝手に追加しない（提案はOK、実装はNG）
- 仕様の用語（due/weak等）はブレさせない（変更提案は別途）

## 3) 変更は小さく安全に（No Big Refactor）
- 1PR/1目的、差分は小さく分割
- 依頼されていない大規模リファクタは禁止
- 既存挙動を壊さない（後方互換を優先）

## 4) 検証ループ（Verification Mandatory）
- 変更ごとに検証方法を必ず提示し、可能なら実行手順を示す
- 最低限：build / lint(typecheck) / test（ある場合）
- 画面がある機能は「手動確認観点」も添える

## 5) UX/コピー（子ども・保護者向け）
- 文言はやさしく、命令口調を避ける
- エラー/不正解でも萎えさせない表現（責めない・次が分かる）
- 迷った時の逃げ道（スキップ等）がある前提で設計する
- 1画面1目的、操作ステップ最小

## 6) 実装スタイル
- 読みやすく、説明できるコード（賢さ優先はNG）
- 型/境界条件/異常系を軽視しない
- 不要な依存追加・過剰な抽象化は禁止

## 7) 進め方（出力テンプレ）
計画や整理が必要な場合は、以下の順で簡潔に出す：
1) 結論
2) 根拠
3) 前提
4) 代替案
5) 不確実性/要確認
