# Done Logs

## Purpose

Done logs store completed work and verification facts.
They are historical records, not current source of truth.

## Format

- Use one monthly file such as `2026-03.md`
- Use [TEMPLATE.md](TEMPLATE.md) for each entry

## Keep It Small

- Summary
- Key files or areas
- Docs/ADR/runbooks touched
- Verification performed
- Remaining risks or follow-ups
