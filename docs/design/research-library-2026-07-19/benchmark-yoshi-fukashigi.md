# 「ヨッシーとフカシギの図鑑」ポップさ再ベンチマーク

- 比較日: 2026-07-19
- 一次資料: [公式トップ](https://www.nintendo.com/jp/games/switch2/aakga/)、[図鑑の中へ](https://www.nintendo.com/jp/games/switch2/aakga/story/)、[生き物を調査](https://www.nintendo.com/jp/games/switch2/aakga/discover/)
- Sansu対象: [v2コンセプト](./concept-keyframe-v2-pop.png)、[v3状態文法](./concept-state-grammar-v3.png)、[Research Library実装](./implementation-completed-390x844.png)
- 目的: 固有IPを模倣せず、公式画像が持つポップさを色、形、材質、カメラ、演技、コピー、編集リズムへ分解し、Sansuが現行案から抜け出す制作契約へ変換する

## 今回の訂正

前回の方向名 **「明るい異変劇場」は撤回する**。

この方向は、暗さ、古紙、疑問符で不可思議さを作らない点までは正しかった。一方で、次の問題を残した。

- 明るいが、色面が上品に調和しすぎている
- 役者も対象も丸く優しく、形の衝突がない
- 全レイヤーが切り紙、濃線、紙ノイズで統一されている
- 相棒が笑顔で指し、対象が静かに光るだけである
- ライブ世界より、本、レバー、カード、進捗が大きい
- ポップさを「安全で明るいこと」までしか定義できていない

その結果、暗い幻想画からは離れたが、**上品なファンタジー教材**に留まった。ユーザーが求める、見た瞬間に触りたくなり、形だけでも少し笑えるポップさには届いていない。

新しい内部アートディレクションを **「いびつ生態ポップ」** とする。

> 算数の答えが、生き物の一芸になる。

補助テストは次の一文である。

> 暗くてもポップ。動かなくても変。

つまりポップさを明度だけへ依存させず、極端な形、焦点色、異素材、二者の演技で成立させる。

## 結論

公式のポップさは、パステル、古紙、かわいい顔の足し算ではない。

```text
一目で読める極端な形
+ 一つへ偏らせた強い色面
+ ライブ立体と鉛筆線の衝突
+ 主人公と対象の二者演技
+ 感覚的で短い結果文
= ポップな発見
```

逆に、次はポップさを弱める。

```text
色を均等に配る
+ 全員を丸く優しくする
+ 全画面を同じ紙質と輪郭で揃える
+ 矢印とカードで因果を説明する
+ 進捗を最初に読む
= AIが整えた学習アプリ
```

## 証拠セット

今回の監査では、同一ラウンドで公式サイトと掲載画像を取得し、次を実見した。

1. モバイルのファーストビュー
2. `生き物を調査` の導入画面
3. 発見結果5場面
4. 明るい場面と暗い場面のギャラリー
5. 開いた図鑑ページ
6. Sansu v2、v3、現行Research Library

公式の掲載画像から読み取れる主な事実は次である。

- ライブ場面は、黄色い地面、空色、緑の主人公など、大きな原色面を持つ
- 暗い場面でも、赤い接触線、黄色い口、緑の葉など、焦点の形と色が強い
- 主人公、生き物、背景、鉛筆残像、本が同じ材質ではない
- 巨大な植物、木、口、群れが画面外へ切れ、世界が枠内へ収まらない
- 生き物は、球、筒、極細脚、巨大口、積層、左右非対称など、個体ごとに輪郭語彙が違う
- 発見結果は、説明ではなく、味、気絶、運搬、変色、成長などの具体的な身体事実である

## 簡易画像計測

公式PNG 9枚を同じ処理で測った。輝度はRec.709、彩度はHSV、密度ピーク比は画面を横8帯へ分けたときの最密帯 / 最疎帯である。閾値は高彩度 `S >= .60`、明部 `L >= .75`、暗部 `L < .25` とした。これは美術の合否を自動決定する値ではなく、目視差を反証可能にする補助である。

| 公式画像 | 輝度中央値 | P90−P10 | 高彩度面積 | 明部 | 暗部 | 密度ピーク比 |
|---|---:|---:|---:|---:|---:|---:|
| Hero | .84 | .39 | 12% | 70% | 1% | 3.0倍 |
| 発見1〜3 | .69〜.72 | .48〜.49 | 74〜75% | 38〜43% | 2% | 15.2〜16.4倍 |
| 発見4 | .48 | .58 | 58% | 23% | 4% | 3.0倍 |
| 発見5 | .83 | .50 | 39% | 64% | 5% | 16.3倍 |
| 図鑑 | .74 | .61 | 17% | 49% | 4% | 2.8倍 |
| 暗いgallery 3 | .44 | .51 | 47% | 4% | 19% | 5.3倍 |
| 暗いgallery 7 | .35 | .39 | 28% | 5% | 28% | 3.5倍 |

色相面積も虹色均等ではない。発見1〜3は暖色面約73% + 黄緑〜緑約19%、発見4は暖色80% + 緑16%、暗いgallery 3は赤〜橙76%、gallery 7は青〜cyan59% + 黄〜緑29%である。Heroは高彩度12%、図鑑は17%しかなく、ポップさと全面高彩度は同義ではない。

Sansu側を同じ系統のheuristicで比較すると、v2は暗部27.5%・edge密度30.5%で、暗さと細部が全域へ広がる。v3ライブ面は高彩度53.9%だがcyanが52%へ全域化し、Research Library実装stageは暗部22.0%でcyan UI枠が画面を支配する。つまりSansu v3は彩度不足ではなく、**色・輪郭・素材・密度を均等に配ったため、跳ねる場所がない**。

重要なのは、公式が単に明るいことではない。

- 明るい発見では、**高彩度の大面積**を恐れない
- 暗い発見でも、**大きな単純形と局所色**でポップさを保つ
- 一つの色相へ意図的に偏らせ、残りの色を役者と反応へ使う
- 全領域を均等に細かくせず、役者の周囲だけを読ませる

Sansu v3は彩度の高いteal、moss、cream、coralを全域へ均等に配り、同じ中間明度へ揃えたため、**色は多いのに焦点がない**。

## 公式のポップさを作る8つの文法

### 1. 色は調和より「偏り」

公式は、色をセマンティックtokenのように均等配布しない。

- 一場面の背景は一つの色族が支配する
- 主人公は背景から外れる固定色を持つ
- 対象は主人公とぶつかる一色を持つ
- 接触や反応には、さらに小さい一色を使う
- 同面積のteal / coral / amber / greenを並べない

黄色い世界へ緑と桃を置く、暗青の世界へ黄と赤を置くなど、**全体の美しい調和より、焦点での色衝突**を優先する。

### 2. 形は「かわいい平均」ではなく極端比率

公式の生き物は、全員が同じ丸顔、大きな目、頬、短い手足ではない。

- 球体に細すぎる脚
- 長い筒に小さすぎる顔
- 巨大な口に小さい胴
- 重なった雲や葉の群れ
- 左右で高さや重心が違う体

顔ではなく、輪郭だけで覚えられる。一体につき一か所、**少し格好悪いほどの比率差**を残すことが人為的な癖になる。

### 3. 世界はフレームへ収めない

公式のライブ世界では、巨大な木、植物、洞窟、身体の一部が画面外へ切れる。

- 大きな前景が端から侵入する
- 地形は画面外へ続く
- 主人公と対象の間だけは作用のために空ける
- 世界を角丸カードや濃い洞窟枠へ閉じ込めない

Sansu v3は、すべてを丸角の完成画へ収めたことで、ゲームではなく展示パネルになった。

### 4. ライブ世界に紙フィルターを掛けない

公式は、複数の材質を一つの画風へ均さない。

| 役割 | 材質 |
|---|---|
| 主人公 | 柔らかく立体的、輪郭が安定、色が強い |
| 生き物 | 個体固有。毛羽、濡れ、硬さ、溝、半透明など |
| 背景 | 大きな描き割り面、粗い筆致、低いedge密度 |
| 観察 | 鉛筆の過去姿勢、短い線、記号 |
| 図鑑 | 初めて全面が紙になり、彩色図と線画が混在 |

現在と記憶の素材が衝突するから、観察が魔法のフィルターではなく「理解した瞬間」に見える。

### 5. 演技は一方向ではなく「掛け合い」

必要なのは、相棒が操作し、対象が光ることではない。

```text
相棒が仕掛ける
→ 対象が予想外に返す
→ 相棒の姿勢まで崩れる
```

対象の反応だけでなく、相棒のカウンターポーズまでが一つのオチである。指差し笑顔は説明係の姿勢であり、ライブ場面の標準姿勢にしない。

### 6. コピーは進捗より「身体事実」

公式の結果文は、何ページ進んだかではなく、いま何が起きたかを短く言う。

Sansuも第一コピーを次へ変える。

- NG: `3つの てがかりが つながった`
- NG: `ほたる花の ひかり道を みつけた`
- OK候補: `ねっこが 先にいっちゃった！`
- OK候補: `ひかりが じめんを はしった`

手掛かり、ページ、保存、完了は、その身体事実を見せた後の第二情報にする。

### 7. 密度を一定にしない

公式のリズムは次である。

```text
密なhero
→ 疎な対面
→ 一点の接触
→ 大きな反応
→ 不均等な図鑑編集
```

同じ角丸、同じ枠、同じ余白、同じ彩度が続かない。人が編集した感じは、統一感だけではなく、**意図的な不均一**から生まれる。

### 8. UIは反応のあとへ下がる

公式の発見場面では、世界と役者が画面を支配する。図鑑へ移った時だけ、紙面が主役になる。

Sansuでは回答中だけ問題棚を出し、保存成功後はstageを約80%へ広げる。ライブ反応の上へ、カード、進捗pill、レバー、矢印を重ねない。

## 新アートディレクション「いびつ生態ポップ」

### ポップ予算

一フレームで強くしてよい軸は、色、形、縮尺、表情、素材、密度、動勢のうち最大3つとする。

- 通常遭遇: 色相差 + 縮尺差 + 一本の動勢
- 観察: 素材差 + 表情差 + 密度差
- 図鑑: 余白差 + 文字差 + 標本縮尺差

彩度、太線、発光、大口、巨大化、粒子、装飾文字を同時に最大化すると、ポップではなく生成画像的な騒がしさになる。公式は毎画面を派手にしているのではなく、画面ごとに2〜3軸だけを急激に跳ねさせている。

### 一画面の原則

```text
一画面一生物
一生物一極端形
一発見一つの接触
一反応一つのカウンターポーズ
一記録一つの感覚的事実
```

### 時間の比率

- 静かな観察 70%: 動かなくても形が妙
- 小反応 20%: 接触へ一つ返す
- 大発見 10%: 世界規模の変化

平常時を低彩度で無個性にしない。静かさは、彩度を落とすことではなく、動く箇所と情報量を絞ることで作る。

### 色契約

通常の明るい生息地では、次を出発点にする。

- 背景の主色族: stageの55〜70%
- 第二色族: 20〜30%
- 対向する焦点色: 3〜8%。相棒と対象の固有色を含めても、主色族と等面積へしない
- 高彩度pixel: 通常遭遇35〜65%、発見ピーク45〜70%、図鑑12〜22%を出発点にし、状態ごとに面積を変える
- 明部: 45%以上
- 暗部: 8%以下。明るい生息地では5%以下を目標にする
- 相棒または対象と直後の背景の相対明度差: 0.18以上
- 鮮色の役割: 最大3系統

暗い生息地は例外として許可する。ただし、深色を外周フレームへ使わず、単純な焦点形、局所色、作用軌道を保つ。

### 形契約

- 生き物は `丸い / 長い / 積層 / 楔形 / 枝分かれ` のいずれかを主語にする
- 一つ以上の寸法比を2.5:1以上にする
- 1体につき、少し格好悪い特徴を一つ残す
- 左右対称の花、顔、体を標準にしない
- 顔は最小2点でよい。大きな目、頬、固定笑顔を必須にしない
- 相棒と対象で同じ輪郭語彙を使わない
- 64pxの黒一色で個体と姿勢を識別できなければ不採用

### 材質契約

- ライブ世界のglobal parchment overlay: 0
- 背景: 面の大きいガッシュまたはマーカー。閉じた濃線を使わない
- 相棒: 2〜3色のくっきりした面。全姿勢で比率を固定する
- 対象: 一種につき一つの固有材質を持つ
- 観察: 同じsceneへ鉛筆残像1〜2件を侵入させる
- 図鑑: 初めて全面を紙へする
- ノイズ: 素材ごとに一種類、3〜5%。全画面共通overlayは使わない

### 演技契約

1. `notice`: 相棒と対象が互いを認識する
2. `commit`: 算数動詞が対象へ直接届く
3. `counter`: 対象が予想外の反応を返す
4. `off-balance`: 相棒の全身が巻き込まれる
5. `hold`: 同じcameraで結果を読む
6. `observe`: 過去姿勢だけが鉛筆線で重なる

口と目だけを変えない。重心、手足、葉、胴体、接地位置まで変える。

## 相棒は再選定する

現行のクリーム色は候補として残せるが、涙形、点目、頬、固定笑顔、二枚葉の組み合わせは完成キャラクターと見なさない。

次のvisual gateでは、同じ算数能力を持つ相棒を **正確に3案** 作る。

各案に必要なもの:

1. 64px黒シルエット
2. `idle / notice / commit / off-balance / recover / celebrate` の6姿勢
3. 算数が身体能力として見える一つの仕組み
4. 固定色2〜3色
5. 対象と衝突した時の崩れ方

手で指差さない。固定笑顔を標準にしない。口は強い反応時だけ現れてよい。

## root-tangleを最初の証明にする

現行の左右対称の宝石花、レバー、発光矢印はvisual targetから外す。ドメインIDと因果データは維持してよい。

新しい一文ルール:

> 数だけ根の輪をほどくと、残った輪に足が生えて、相棒より先に走っていく。

必要な一場面:

1. 低い重心と長すぎる根を持つ、非対称な対象
2. 相棒が指定数の輪へ直接触れる
3. 残った輪だけが別材質の小さな足へ変わる
4. 灯りの列が相棒を追い越す
5. 相棒が一拍遅れて安全に尻もちをつく

第一コピー候補:

> ねっこが 先にいっちゃった！

観察文候補:

> のこった輪が、ひかりの足になった。

図鑑では、走る根を最大図にする。操作前、追い越し、尻もちを小図にし、等幅三カードへしない。

## 制作工程

### いま止める

- 現行画風への色足し、キャラクター足し、装飾足し
- v2 / v3をproduction visual targetとして使うこと
- `明るい異変劇場` を現行方向名として使うこと
- 三状態をまとめた新しいコンセプトボード
- live actorをinline SVG、CSS図形、絵文字で仮置きすること
- model sheet承認前のsprite量産

### 次のvisual gate

同じ `root-tangleの追い越し直前`、同じ390×844、同じ情報量で、正確に3案だけ作る。

各案は次だけを含む。

- 一つの単一時点フルカラーframe
- 相棒と対象の黒シルエット
- 使用色4役
- 材質4役
- どこが少し格好悪いかの一文
- 何が相棒へ返ってくるかの一文

ユーザーが一案を選ぶまで、本番コード、相棒rig、sprite、追加画面へ進まない。

### 選定後

1. 相棒6姿勢model sheet
2. 対象のbefore / contact / counter / settle behavior sheet
3. camera descriptor
4. stage背景と役者の実asset
5. observation用鉛筆差分
6. 図鑑用大小vignette
7. 390×844実装と同一camera検証

[`agent-sprite-forge`](https://github.com/0x0funky/agent-sprite-forge) は、承認後の透過、整列、atlas化、差分検査にだけ使う。独自性、シルエット、演技を決める工程には使わない。

## IP境界

### 借りる原則

- 極端な単純形を一体ずつ立てる
- 一つの接触と一つの身体反応
- ライブ立体と観察線の素材差
- 世界をフレームへ閉じ込めないカメラ
- 感覚的な結果を先に伝える
- 不均等な図鑑編集

### 借りない表現

- ヨッシーの輪郭、鼻、甲羅、背びれ、配色、舌、タマゴ、固有アクション
- 公式生き物の形、顔、材質、群れ方、身体ネタ
- 顔や虫眼鏡を持つ本
- 公式ロゴ、書体、星、卵、ページ装飾、具体的レイアウト
- 公式画像のtrace、crop、camera、animationの再現
- 既存IPへ似ていることを合格理由にすること

Sansuの固有性は、**数量変化が生き物の一芸になり、その反応が相棒へ返ってくること**に置く。

## 合格ゲート

### 形

- 64px黒シルエットで相棒と対象を識別できる
- 生き物の一寸法が2.5:1以上へ意図的に偏る
- 相棒と対象を汎用スマイルへ交換すると演技が壊れる

### 色

- 背景主色、相棒、対象、接触の役割が説明できる
- セマンティック色を均等配布していない
- 暗いvariantでも焦点境界の相対明度差0.18以上を保つ

### 演技

- 文字なしで `仕掛け → 返し → 相棒の崩れ` を並べられる
- 対象の反応だけでなく相棒のcounter poseがある
- 指差し、矢印、放射線を消しても因果が読める

### 材質

- 背景、相棒、対象、観察線の4役が白黒でも区別できる
- ライブ世界へglobal paper overlayがない
- 図鑑だけが全面紙になる

### 画面

- ライブstageが画面幅いっぱいで、汎用カードへ入っていない
- 発見ピークの汎用カード数は0
- 回答棚の汎用surface数は1
- 一場面一時点で、三状態を並べていない

### コピー

- 第一文が感覚、身体、音、物理変化のいずれかを述べる
- 第一文に `ページ / 手掛かり / 進捗 / 完了` を使わない
- 一行18全角文字以内を基本にする

### アクセシビリティと拡張

- 主操作56px、補助44px、意味を持つ文言12px以上
- sound-off、reduced motion、200%文字拡大でも因果と続行操作が残る
- 生き物を追加しても画面componentへ個体名分岐を増やさない
- art不一致でも閉じられるDOM fallbackから進行できる

## 現時点の判断

Sansuのゲーム構造、保存receipt、責めない再試行、同一cameraの観察方針は残す。

しかし、現行のvisual shellは残さない。

- v2: 失敗例。暗く、細かく、世界が装飾で埋まる
- v3: 状態順だけ残す。画風、相棒、花、レバー、図鑑編集は不採用
- Research Library実装: データ、DOM順、主操作、ARIAは残す。深いteal枠、均一切り紙、進捗pill、小相棒は作り直す
- `明るい異変劇場`: 撤回。現行SSOTから `いびつ生態ポップ` へ置換する

次の評価対象は、画面全体の完成度ではない。**同じroot-tangleの一時点を描いた、相棒と対象の掛け合い3案**である。

## 証拠限界

今回の比較は、公式サイトと公式掲載静止画を対象にした。画像から音の質、実アニメーションの間、入力の手触り、長時間プレイの疲労、完全なアクセシビリティは断定していない。公式の固有画像は比較根拠にだけ使い、Sansuの成果物へ転用しない。
