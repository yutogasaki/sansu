# Task Docs

## Purpose

Task docs capture active execution context only.

## Split

- `docs/11_full_task_backlog.md`
  - Global backlog and prioritization
- `docs/12_ui_fix_tasklist.md`
  - Domain backlog/tasklist
- `docs/tasks/active/`
  - The small set of tasks currently being executed

## Lifecycle

1. Pull from backlog into `active/`
2. Keep scope small
3. Close by moving durable facts to `docs/done/` and, if needed, `memory.md`
