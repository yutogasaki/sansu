## 3. カラー

探索の静けさは低彩度化ではなく、主因果と可動領域を絞ることで作る。ライブ世界、観察、保存で色と素材の役割を分ける。

| 用途 | 方向 |
|---|---|
| Utility / 保護者・設定 | オフホワイト、淡いミント、淡い水色。ガラス面を許可 |
| ライブ世界 | 一つへ偏らせた不透明な中〜高彩度の大色面。ガラス面を使わない |
| 観察 | ライブ色を残し、変化箇所だけへ観察線を追加 |
| 保存・図鑑 | 紙の余白、主標本1点、大小不同の因果挿絵 |
| 危険 | 赤ではなくオレンジ、揺れ、注意アイコン |
| 成功 | 緑だけに頼らず、光、音、形で表現 |

### 3.1 「いびつ生態ポップ」の画風契約

- 旧目安「不可思議な図鑑 70 : ポップ 30」と後続の「明るい異変劇場」は棄却する。前者は暗さと紙を増やし、後者はポップさを高明度と短い身体ギャグへ縮めすぎた
- Sansu固有の式は `極端な形 + 偏らせた焦点色 + 素材の衝突 + 対象からの返し + 相棒の崩れ + 身体事実の一文` とする。数量変化が生き物の一芸になり、その反応が相棒へ返ることを独自性の核にする
- 一画面は3色相族までを出発点にし、`主調色55〜70% / 副調色20〜30% / 対向アクセント3〜8% / 無彩色余白` へ偏らせる。主役色と直後の背景は色相差70°以上を目標にし、テーマ色を等面積で配らない
- 明るい通常遭遇では深い青緑を輪郭・奥行き端へ限定し、画面の約25%未満を目安にする。暗い生息地も許可するが、大きな単純形、局所色、一本の作用軌道を保つ
- 生き物は輪郭の60〜80%を大きな曲線にし、硬い平面・葉・くちばし・裂け目などを一つだけ衝突させる。一寸法を2.5:1以上へ偏らせ、64pxの黒一色で個体と姿勢を識別できなければ不採用とする
- 不可思議さは扉、疑問符、紋章、暗さではなく、対象ごとの「一つの妙な身体規則」で作る。各遭遇は `誰が / 何をすると / 何が返り / 相棒がどう崩れる` を一文で定義する
- ポップ予算は一フレーム最大3軸とする。色、形、縮尺、表情、素材、密度、動勢のうち2〜3軸だけを跳ねさせ、彩度、太線、発光、大口、巨大化、粒子、装飾文字を同時に最大化しない
- 基本cameraは子どもの目線に近い横向き舞台とし、相棒と対象の間へ作用が通る余白を残す。祭壇型の左右対称、見下ろしの説明図、中央集約を標準にしない
- 世界は大きな平面ガッシュ、役者は輪郭と面が安定した承認済みアート、観察は変化箇所だけの濃線と残像、保存はクリーム紙と大小の挿絵へ役割を分ける。全レイヤーへ同じ紙ノイズや縁取りを重ねない
- UIは探検道具、ノート、スタンプとして世界へ統合してよい。ただし操作ラベル、問題、答え、状態はDOMに置き、キャラクターや主要生き物を即席のCSS図形や未監修SVGで代用しない
- 生成・制作時は固定cameraの背景、相棒、対象、道具、作用差分を再利用可能なlayerまたはlocked referenceとして管理する。配信時は、分解合成で絵本調の質感、輪郭、接地、継ぎ目が失われる場合、UI文字を含まない承認済みstate別flattened frameを使う。三状態を一枚へまとめた説明ボードや未承認の一枚絵は本番へ採用しない。相棒は黒一色のsilhouetteと最低6姿勢を承認してからproduction assetへする
- 通常時から固有の輪郭、視線、重心を見せ、`ready → anticipation → contact → reaction → settle` の順で対象と相棒の掛け合いを作る。同じ場面へ観察の痕跡を残し、無個性な待機姿勢へ戻さない
- 390×844では回答前のstageを画面高55〜65%、問題・入力を35〜45%とし、resolved後はstageを72〜82%へ広げることを出発点にする。相棒は高さ116px以上、対象は高さ140pxまたは幅150px以上、作用空間は最低72pxを確保する
- motionの目安は `予備動作 180〜250ms → 接触 150〜300ms → 返し 500〜900ms → 結果保持 900〜1400ms` とする。reduced motionでも状態順と保持時間を消さない
- コピーは身体・感覚・音・物理変化を第一情報にし、ページ、手掛かり、保存、完了などの進捗語を第二情報へ下げる

### 3.2 発見ループの三つの視覚モード

- `世界`: `相棒 + 変化する対象 + 1つの算数動詞 + 対象からの返し` を大きく見せ、汎用カードや装飾を主役にしない
- `観察`: 直前の世界と同じカメラ・主要輪郭を保ち、相棒と現在の対象は色を残す。変化した箇所と直前の行動だけを濃色線、残像、短い因果文へ変換する
- `保存`: 観察結果を大きなクリーム紙面へ大小の挿絵として編集し、等幅カードやフローチャートにせず、原因から大発見を再読できるようにする
- 三モードは同じ仕上げを使い回さない。`生きた色 → 選択的な観察線 → 物理本の編集物` という素材差を報酬として見せる
- ライブの主素材を65〜80%、対象の固有素材を15〜30%、観察線を8%以下の出発点とする。紙、光沢、布、粒子、発光を同面積で混ぜない
- ライブstageは画面幅いっぱいへ開き、背景物を3辺以上へ接触させる。主役者と主対象は作用が読める範囲で完全表示し、全オブジェクトを余白付きカードへ収めない
- 横8帯で見た最密帯と最疎帯の密度差は通常3倍以上、遭遇ピーク5倍以上を目安にする。草、記号、模様を全帯へ均等に散らさない
- 390px幅の静止画を3秒見たとき、子どもが「だれが動くか」「何へ何をしたか」「どんな妙な反応が起きたか」「次にどこを押すか」を指せることをレビュー条件にする
- 比較根拠、IP境界、実装優先順位は `docs/design/research-library-2026-07-19/benchmark-yoshi-fukashigi.md` を参照する
- 絵主導candidateは、承認済み絵本調benchmarkをruntime screenshotの横へ置いて比較する。視覚的磁力は画像だけを6軸で採点し **52 / 60以上かつ各軸8 / 10以上**、無文字理解は未説明の5人中4人以上が同じ主動詞とpayoffを答え4人以上が続きを望み、痛み・拘束・切断・武器・恐怖などの危険解釈0件を必須とする。これらとruntime整合は相互に補償せず、一つでもHOLDならproductionへ進めない

## 4. モーション

旧仕様では派手な拡縮や勝利演出を抑えていた。新仕様では、常用しない範囲で決め場の派手さを許可する。

許可:

- 岩破壊の短いポップ
- 水晶の連鎖グロー
- 発見時の軽いカメラズーム
- 橋がぷるぷる揺れる
- 帰還時の気球・地図アニメーション
- 連続正解時のリズムあるSE

禁止:

- 長い演出でテンポを壊す
- 点滅
- 子どもを煽る敗北演出
- 常に画面がうるさい状態
- 正誤を色だけで伝える

連問テンポの上限:

- 通常正解保持: 180ms以下
- 専用遭遇の正解保持: 320ms以下、発見開始待ち80ms以下
- 同一区間の正解から次問操作可能: 目標650ms以内、追加0タップ
- 不正解から再入力可能: 550ms以下
- 通常発見・通常調査手掛かり: 約900ms、非blocking
- 保存中表示: 250msを超えた場合だけ表示

アクセシビリティ:

- `prefers-reduced-motion` では反復移動、揺れ、回転、拡縮を停止する
- 正誤、残りエネルギー、発見はテキストまたはARIAでも伝える
- 問題回答中はテンキーをネイティブに無効化し、二重送信を防ぐ
- ロック中のノードは中身を示す絵柄ではなく、未知の記号で表示する

### 4.1 ひかりの橋

- 未入力: 左右の光は中央の水晶へ届く直前で止め、「まだ つながっていない」とテキストでも伝える
- 誤答: 橋を壊さず一度だけ短く揺らし、「橋が ぽよん」とエネルギー消費を通知する。その後は同じ遭遇内へ具体物ヒントを残す
- 正解: 入力を短くロックし、同じ `cameraKey / viewBox / anchor` 内で紙リボン、相棒、岸の `stateVariant` だけを未完成から完成へ変える。同一区間では相棒の通過と次問準備を同時進行させ、別の続行操作や長い静止フレームを挟まない
- `prefers-reduced-motion`: 揺れ・拡縮・移動補間を止め、同じ構図の静的な状態差し替えと成功文言だけを使う。情報は省略しないが、固定待機を増やさない
- scene artへ問題、答え、HUD、キーを焼き込まず、動的情報と操作はDOMに保つ

### 4.2 根っこのからまり

- 未入力: やわらかい根が花の抜け道を編み目状に塞ぎ、「まだ からまっている」と伝える
- 誤答: 根を傷つけず短く「くるん」と戻し、同じ遭遇内へ引き算の具体物または10のまとまりを残す
- 正解: 相棒の数リボンが、引く数だけ根の輪を同じcamera内でほどく。残った輪の個数を視認できる時間を置く
- 反応: 残った根の輪が小さな灯りの足へ変わり、相棒を追い越して一列で歩き出す。第一コピーは「ねっこが 先にいっちゃった！」、観察文は「のこった輪が、ひかりの足になった。」を出発点にする
- 結果: 灯りの列が抜け道を作り、追いかけた相棒が一拍遅れて安全に尻もちをつく。子どもの位置と主要地形を保ち、数量変化と通過結果を同時に読める時間を残す
- asset契約はファイル名や全面JPEGではなく、`cameraKey / viewBox / anchor / layer / pose / stateVariant` で定義する。runtimeに残る暫定JPEGはcamera continuity検証用であり、画風、構図、cropの規範にしない
- 観察: 結果sceneの同じcamera、主要輪郭、object position、bounding box、相棒と対象の色を維持し、変化経路だけへ承認済みの鉛筆差分と短い因果文を重ねる。全画面の低彩度化、紙filter、即席SVG stampで代用しない
- 最終assetは承認済みmodel sheet由来の相棒・対象layer、広い背景plate、観察差分へ分ける。外部URLへ依存せず、PWA offlineで同じcamera descriptorから解決できること
- 棘、切断、締め付け、敵、怖い暗さは使わない

### 4.3 専用遭遇の制作契約

- 未完成、完成、結果は同じカメラ・地形・主要物の構図を維持し、変化の因果を読み取れるようにする
- scene descriptorは `cameraKey / viewBox / anchor / layer / pose / stateVariant` を持ち、ファイル名や全面rasterの暗黙cropを状態契約にしない
- 誤答支援は別画面へ飛ばさず、同じ遭遇内に数量の模様・具体物・短い次アクションを残す
- 画像、音、モーションが読み込めなくても、問題・状態文言・入力・続行操作はDOMだけで成立させる
- 専用遭遇は `loading / ready / incorrect / correct / resolved` の共通フェーズを使い、ページ固有の長い状態機械を増やさない
- `prefers-reduced-motion` では観察へのfilter transition、stamp pop、移動、zoomを止める。世界反応は短い状態更新へ置換し、通常発見で続行操作や固定待機を追加しない
- 正解保持時間と発見開始時間は遭遇定義に置き、原因と世界変化が読める時間を確保する
- 画像はリポジトリの `public/assets/explore/<encounter-id>/` に保存し、リモート開発・PWA・共有時も同じURLで参照できるようにする

### 4.4 cold-open導入区間

delivery / feature-flag ID `snap-root-v1` のlocal validationへ載せる現行visual candidateを `dig-pop-painted-v2` とする。二つのIDは別々に記録し、同じslotを使った旧candidateの承認や点数を継承しない。390×844と768×1024で同じ相棒、スコップ、土、根生物、cameraを保ち、次の状態を問題の前後で連続して読ませる。

- commit済み正解数だけを `ready / dig-one / dig-two / popped` へ写す
- `ready`: 葉帽子の相棒がスコップを持ち、同じ大きな根生物が土へ半分埋まった状態を予告する。相棒と根生物は互いを見る
- `dig-one`: 1問目の直後に相棒が根生物の身体や葉へ触れず、その周囲の同じ土を掘る。土塊が飛び、問題UIを隠さず次問準備と同時進行する
- `dig-two`: 2問目の直後に同じ場所の土がさらに減り、根生物が上へ持ち上がって両足が見える。`dig-one` との差を、粒子追加だけでなく地面線、対象高、足の露出で読ませる
- `popped`: 3問目に同じ根生物が全身でぽんと抜けて安全に接地し、相棒は全身が連続した尻もち姿勢になる。柔らかい土塊が相棒の葉帽子へ載り、両者の無傷・接地・回復を文字なしで読めるようにする。光、粒子、巨大文字、進捗バッジだけでオチを代替しない
- 1・2問目は追加0タップで、正解から次の入力可能まで650ms以内。不正解はstageを変えず、550ms以内を目標に同じ問題へ戻す
- 問題の正答と一致する個数、長さ、目盛り、序列をsceneへ置かない。答え漏洩は0件を必須とする
- 固有名と永続図鑑ページはcold-openへ追加しない
- `prefers-reduced-motion` では移動補間、土の飛散、跳ね、回転、zoomを止めても、4状態の地面線、対象高、足の露出、最終位置、原因と結果を静止差分で維持する
- 旧編み根、`nest-squash | nest-tip`、旧 `landed`、根生物の頭の葉を相棒が引く `tug / tumble` は撤去する。実機画像で、相棒が編み輪に締め付けられた身体、根生物が浮遊する切断頭部、一本葉が舌・腸・蛇や引っ張られる髪へ見える危険が確認されたため、安全な比較variantとして再利用しない
- payoffはコピーを隠した64pxでも相棒と根生物の全身が連続し、土や葉が首・顔・胸を横切らず、スコップが身体へ向いていないことを必須とする
- 旧source candidate `dig-pop-painted-v1` は視覚的磁力 **50 / 60** のHOLDとする。背景、相棒、根生物、スコップの形がstate間でdriftし、`dig-one → dig-two` の差が弱かったため、現行runtimeやv2の承認根拠として扱わない
- `dig-pop-painted-v2` は一枚のlocked backgroundと承認済みcharacter / prop referenceへ作用差分だけを重ね、実problem panelへ配線した。runtime再採点は390×844で **52 / 53 / 52 / 53**、768×1024で **52 / 53 / 53 / 53**、全6軸8以上となり視覚ゲートを通過した。最初のtablet `dig-two` はanswer shelfが両足とスコップ刃を隠して **50 / 60** のHOLDとなったため、背景を変えずaction overlayを上へ再配置し **53 / 60** へ修正した。顔、作用する手・道具先端、対象の接地・露出した足、payoff propはすべてのcanonical cropでanswer-shelf fadeより上に保つ

#### `dig-pop-painted-v2` authoring / flattened delivery

- authoring sourceは一枚のlocked background、承認済みの相棒・根生物・スコップreference、地面線・土塊・姿勢のstate overlayへ分ける。同一characterをstateごとにゼロから再生成せず、camera、比率、顔、葉、道具、光を固定する
- deliveryは絵本調の土、葉、空、身体の材質と継ぎ目を守るため、UI文字なしのmobile 800×1000とtablet 1024×768のflattened JPEGをそれぞれ `ready / dig-one / dig-two / popped` の4枚にする。`<picture>` は現在viewportの4枚だけを先読みする。各frameは180KiB以下を目標とし、問題、答え、進捗、TenKey、状態文言はDOMへ残す。layered authoringは必須だが、runtimeを分解SVGへ戻すことは要件にしない
- visual candidate ID、stage、actor、対象、action、asset load状態、reduced motionはDOMの安定属性とARIA descriptionで検査でき、画像内文字やファイル名だけを判定根拠にしない
- payoff variantは作らない。最初の安全証明は一つのcanonical `popped` で行い、別オチは視覚的磁力、身体完全性、silent testを別々に通過した後だけ追加する
- reduced motionでは承認済み静止frameを即時差し替え、transition、土の飛散、跳ね、回転、zoomを止める。状態や最終位置は省略しない
- production defaultは `classic-v1` とし、local validation candidate `dig-pop-painted-v2`、source HOLDの `dig-pop-painted-v1`、一本葉を引くBloom版、水やり版、`root-pull-v1` / `root-pull-v2` は現行defaultから外す。既存の引き算専用遭遇 `root-tangle`、そのreceipt、観察、保存IDとは共有しない

## 5. 音

音はゲーム性と掛け合いを支える。genericな正解音を鳴らすだけでなく、視覚の因果と同じ順序を持たせる。

- `anticipation`: 触れる直前の小さな予告。連打せず、主動線を一つに絞る
- `contact`: 算数動詞が対象へ届いた瞬間の短い材質音
- `reaction`: 対象固有の返し。生き物ごとに音高、長さ、間を変える
- `comic beat`: 相棒が巻き込まれる直前または直後へ一拍の空白を置く
- `settle`: 結果を読ませる短い着地音。ジングルで身体ギャグを上書きしない

- 正解: 軽い「カチッ」「キラッ」
- 岩破壊: 気持ちいい破砕音。ただし怖くしない
- 発見: 短いジングル
- 連鎖: 音階が上がる
- 失敗: 低く責める音ではなく、コミカルな「ぽよん」

音OFF設定は維持。

## 6. いきもの

いきものは探検相棒へ統合。

役割:

- 探索開始時に一言
- 発見時にリアクション
- 不正解時に責めずにヒントへ誘導
- 帰還時に今日の物語をまとめる

NG:

- 「まちがえたね」
- 「もっとがんばろう」連発
- 失望表情

OK:

- 「ちがう道から見てみよう」
- 「岩にヒントのもようが出たよ」
- 「あと一歩で届きそう！」

## 7. 失敗演出

失敗は面白く、短く、再挑戦したくなるものにする。
