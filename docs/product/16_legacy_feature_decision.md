# docs/product/16_legacy_feature_decision.md — 旧機能の維持・統合・廃止一覧

> 状態: 旧ホームは通常起動面から外し、`/` を `/explore` へ転送する。`/study`、いきもののデータ、バトル、設定は維持する。旧「ふわふわ」アルバムはデータを削除せず、探検相棒・発見ノートへの視覚統合が終わるまで子どもの記録画面から隠す。

## 1. 維持

| 機能 | 理由 |
|---|---|
| 算数118スキル | 学習資産の中核 |
| 問題生成 | 探索のアクション源になる |
| SRS / Due / weak | 学習継続の裏側に必要 |
| テンキー・複数入力・筆算UI | 既存資産。探索問題パネルで使う |
| ローカル保存 | 現行思想とPWAに合う |
| 保護者設定 | 信頼のため必要 |
| 定期テスト | 探索とは別の測定モードとして残す |
| `/dev` | 問題・探索デバッグに必要 |

## 2. 変更

| 機能 | 変更内容 |
|---|---|
| 親仕様 | 学習アプリから算数学習ゲームへ |
| 体験原則 | グリーンで静かな教材から、楽しいが傷つけないゲームへ |
| ホーム | 旧いきもの画面を通常起動面から外し、`/` を探索開始画面へ接続。`/battle` は退出後の探検基地へ変更し、探索再出発を主役、既存2人ゲームを下位導線にする |
| 通常学習 | 通常起動は探索ランへ移し、直列学習は `/study` で維持 |
| 正誤演出 | ○×評価から、岩・橋・発見の結果へ |
| 不正解 | 無影響ではなく、短期的ゲーム影響あり |
| デザイン | 平常時は見やすく、ピーク時は派手に |
| 旧ふわふわアルバム | 旧画風の混入を避けるため記録画面から一時的に隠す。保存データは維持し、探検相棒・発見ノートとして再統合してから戻す |

## 3. 統合

| 機能 | 統合先 |
|---|---|
| いきもの | 探検相棒、ヒント、リアクション |
| きろく | 学習記録 + 発見ノート |
| 復習 | 探索内のDue / weakゲート、または `/study` 復習 |
| イベントモーダル | 探索イベント、地図イベントへ |
| コンボ | バトルではなく探索の鉱脈・連続正解へ |

## 4. 廃止候補

| 機能 | 理由 |
|---|---|
| 既存バトル主導線 | 探索体験と主目的が分散する |
| 2人用敵HPバトル | 暗い・競争・敗北圧が強い。将来は協力探索へ |
| 30日でいきものがgone | 探検相棒としての愛着と衝突する可能性 |
| ただ静かなデザイン制約 | 面白さのピークを潰す |

## 5. 保留

| 項目 | 判断条件 |
|---|---|
| 英語統合 | 地底探索に入れるか、別ゲームにするか検証後 |
| 横画面専用化 | PWA利用実態と端末確認後 |
| 本格橋物理 | 固定橋イベントが面白いと確認後 |
| バッグ容量 | 低学年の認知負荷を見て判断 |
| 帰還失敗時の損失 | 離脱率を見て調整 |
| 外部ランキング | 原則入れない。家庭内記録程度に留める |
